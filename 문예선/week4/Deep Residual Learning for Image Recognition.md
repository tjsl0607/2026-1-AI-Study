### 개념 정리
1. Backpropagation, 역전파
오차(정답과 결과물, 예측값의 차이)를 줄이는 방향으로 학습하기 위해 내부의 Weight(가중치)들을 수정하면서 학습함.
이는 출력층에서부터 입력층 방향으로 거꾸로 되돌아가며 수정할 값들을 계산하는 방식임.
수정할 값을 계산할 때는 미분값, 즉 기울기를 구해서 사용함. 이 기울기는 가중치를 어느 방향으로, 얼만큼 조절해야 오차가 줄어드는가?를 알기 위해 사용함.
![](https://velog.velcdn.com/images/sssun67/post/76984452-6122-4c1a-873b-8acb4991b629/image.png)

2. Vanishing Gradient, 기울기 소실
역전파 과정에서 기울기를 계산할 땐 각 층의 기울기 값들을 계속해서 곱해나감.
*가중치의 변화값에 따른 최종 오차의 변화값, 즉 기울기를 구하는 방법
아래 사진처럼 서로 약분되듯이 이어진다
![](https://velog.velcdn.com/images/sssun67/post/e9c2d33f-f26f-4668-a8f8-90a4505a8cf7/image.png)

-> 각 층에서 곱해지는 기울기 값이 1보다 작다면 층이 깊을수록 이 값을 수십, 수백 번 곱하게 됨. 즉, 0에 수렴함.
결과적으로 출력층에서 멀리 떨어진 입력층 근처의 초기 레이어들은 기울기 값이 0이 되어버려 가중치 업데이트가 전혀 일어나지 않게 됨. 모델의 기초가 되는 앞부분이 학습되지 않으니, 전체 모델의 성능이 떨어질 수밖에 없음.

3. 기울기 폭발
각 층에서 곱해지는 기울기 값이 1보다 조금이라도 크다
-> 층을 거듭할수록 값이 기하급수적으로 증가
-> 기울기가 너무 커지면 가중치가 비정상적으로 크게 설정됨. 이로 인해 학습 방향을 찾지 못하고 숫자가 터져버리는 오류 발생

4. 최적화
가장 오차가 적은 완벽한 가중치를 찾아가는 과정

### 문제 제기
문제: 모델이 깊어질수록 성능 저하 발생
- 신경망 층이 깊어질수록 더 복잡하고 유의미한 특징을 추출 가능 -> 특정 깊이 초과시 테스트 오차, 학습 오차가 오히려 증가
원인:  층이 깊어짐에 따라 최적화가 매우 어려워짐 -> 학습 자체가 제대로 이루어지지 않음. (기울기 소실/폭발 문제 포함)
* 파라미터가 많아져서 생기는 과적합 문제가 원인이 아님
### 핵심 해결책
##### 기존 방식: 입력값을 목표값으로 직접 매핑
어떤 레이어가 학습해야 할 이상적인 매핑 함수 = H(x)
(x는 입력값)
이 때, 기존 신경망 H(x)를 직접 학습하려 함 
#### 잔차 학습(ResNet)
F(x) = H(x) - x를 학습
-> 최종 출력은 F(x) + x가 됨.
*수학적 이해를 위해 https://youtu.be/Fypk0ec32BU 보기
- 스킵 커넥션 (Shortcut / Skip Connection)
잔차학습을 구현하기 위해 x를 연산 중간에 건너뛰어 출력 부분에 그대로 더해줌(Identity mapping)

### 효과
1. 학습의 난이도 하락
단순히 F(x)를 0으로 만들어 추가적 학습을 막을 수 있음.
변화량만 학습하면 되기 때문에 최적화가 훨씬 쉬워짐.
2. 원활한 기울기 전달
역전파 과정에서 스킵 커넥션을 통해 기울기가 깊은 층에서 얕은 층으로 손실 없이 바로 전달 가능

### 결과 및 성과
1. 기존(VGGNet)은 19개 층 수준/ ResNet은 152개 층
2. 대규모 이미지 인식 대회(ILSVRC 2015)에서 인간의 인식률을 뛰어넘는 3.57%의 에러율로 1위를 차지함.
